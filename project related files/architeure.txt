🔗 3️⃣ SMART CONTRACT DESIGN
🔹 Role Storage
mapping(address => Role) public roles;

Admin registers users into blockchain.

🔹 Batch Structure
struct Batch {
    string batchId;
    string parentBatchId;
    address owner;
    string productName;
    string farmLocation;
    string certificateHash;
    uint totalQuantity;
    uint remainingQuantity;
    string unit; // KG or Litre
    uint timestamp;
    bool exists;
}
🔹 Storage Mappings
mapping(string => Batch) public batches;
mapping(string => string[]) public childBatches;
string[] public allBatchIds;
🔄 4️⃣ COMPLETE DATA FLOW
🔹 STEP 1 – Admin Registers User

Admin stores user in MySQL

Admin calls:
registerUser(wallet, role)

Blockchain now recognizes wallet role.

🔹 STEP 2 – Farmer Creates Batch

Farmer:

Uploads certificate → IPFS → gets hash

Calls createBatch()

Smart contract:

Generates batchId automatically

Stores batch

remainingQuantity = totalQuantity

Adds to allBatchIds

🔹 STEP 3 – Distributor Purchase

Distributor selects batch.

Calls purchaseBatch(parentBatchId, quantity)

Smart contract:

require(remaining >= quantity)

parent.remainingQuantity -= quantity

generate child batchId

create child batch

childBatches[parent].push(childId)

Stored permanently.

🔹 STEP 4 – Retailer Purchase

Same logic.

Child becomes parent for next stage.

Tree grows.

🔹 STEP 5 – QR Generation

Retailer generates QR containing:

Final batchId

🔹 STEP 6 – Consumer Trace

Frontend:

Fetch batch details

While parentBatchId != "":
fetch parent

Display full chain timeline

All from blockchain.

🔐 5️⃣ SECURITY MODEL

MetaMask signs all blockchain transactions

Smart contract checks msg.sender role

Only owner can resell batch

require statements enforce quantity rules

No private keys stored in DB

Blockchain immutable history

🔄 6️⃣ BATCH ID GENERATION

Smart contract auto-generates:

Example:

Root: B1
Child: B1-1
Grandchild: B1-1-1

Uses internal counter.

Frontend never generates ID.

📦 7️⃣ QUANTITY RULES

Quantity stored as uint

remainingQuantity updated via transactions

Always latest state returned

Multiple distributors allowed

Sequential blockchain execution ensures safety
